ANOMALY DETECTION IN PAYROLL TRANSACTIONS - PROJECT SUMMARY
================================================================================

Project Location: /sessions/jolly-zealous-archimedes/mnt/anomaly_detection/

OVERVIEW:
A complete, production-ready machine learning pipeline for detecting anomalous
transactions in payroll systems using unsupervised learning. Designed as a
portfolio project for a Staff Data Scientist role at Paylocity (HCM/payroll
software company).

DATASET:
- PaySim Synthetic Financial Transactions (Kaggle)
- 6.3M transactions, ~0.13% fraud rate (highly imbalanced)
- Features: transaction type, amount, balance changes, origin/destination info

PROJECT STRUCTURE:
├── 00_data_collection/      Download, clean, and upload PaySim data to S3
├── 01_eda/                  Exploratory analysis (8 detailed visualizations)
├── 02_preprocessing/        Feature engineering, scaling, temporal split
├── 03_isolation_forest/     Isolation Forest model with Optuna tuning
├── 04_lof/                  Local Outlier Factor model with Optuna tuning
├── 05_comparison/           Side-by-side comparison and ensemble analysis
├── requirements.txt         Python dependencies (13 packages)
└── README.md               Comprehensive documentation

KEY FEATURES:

1. Data Collection (00_data_collection)
   - Download from Kaggle using kagglehub
   - Type conversion and null handling
   - Upload to S3 for reproducibility
   - Summary statistics (fraud rate, transaction types, amounts)

2. EDA (01_eda)
   - Class distribution visualization (99.87% vs 0.13% fraud)
   - Transaction type distribution
   - Amount analysis (histogram with log scale)
   - Box plots by transaction type
   - Fraud rate by type (TRANSFER and CASH_OUT highest)
   - Balance change scatter plots
   - Correlation heatmap
   - Temporal trends (steps over time)

3. Preprocessing (02_preprocessing)
   - Feature Engineering:
     * balance_change_orig = newbalanceOrig - oldbalanceOrg
     * balance_change_dest = newbalanceDest - oldbalanceDest
     * amount_to_balance_ratio = amount / (oldbalanceOrg + 1)
     * is_balance_zeroed = 1 if newbalanceOrig == 0 else 0
     * hour_of_day = step % 24
     * One-hot encoding of transaction type
   - StandardScaler normalization
   - Temporal train/test split (80/20 on steps)
   - Training on non-fraud data only (simulates real deployment)

4. Isolation Forest (03_isolation_forest)
   - Hyperparameter tuning via Optuna (50 trials)
   - Grid: n_estimators, max_samples, contamination, max_features
   - Evaluation metrics: ROC-AUC, PR-AUC, Precision, Recall, F1
   - Visualizations: score distribution, PR curve, ROC curve, confusion matrix
   - Model serialization with joblib

5. Local Outlier Factor (04_lof)
   - Density-based anomaly detection (complements Isolation Forest)
   - Hyperparameter tuning: n_neighbors, contamination
   - Same evaluation framework and visualizations
   - Novelty mode for production inference

6. Model Comparison (05_comparison)
   - Side-by-side ROC curve overlay
   - Precision-recall curve comparison
   - Metrics bar chart (6 dimensions)
   - Score distribution comparison
   - Ensemble analysis: both models must agree (high precision mode)
   - Saved metrics table (CSV)
